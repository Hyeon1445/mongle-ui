변경사항을 새 브랜치에 푸시하고 PR을 생성합니다.

브랜치 이름은 인자로 전달됩니다: $ARGUMENTS

아래 순서대로 진행하세요:

## 1단계: main 동기화

main으로 체크아웃하고 최신 변경사항을 풀받습니다:

```
git checkout main && git pull origin main
```

## 2단계: 브랜치 생성 및 파일 검토

새 브랜치를 생성하고 체크아웃합니다:

```
git checkout -b $ARGUMENTS
```

그런 다음 `git status`를 실행해서 변경된 파일 목록을 확인합니다. 변경된 파일 목록을 사용자에게 보여주고, 어떤 파일을 커밋에 포함할지 확인을 받습니다. 사용자의 승인을 받은 후에 다음 단계로 진행합니다.

## 3단계: 스테이징 및 커밋

사용자가 포함할 파일을 확인해주면, 해당 파일들만 스테이징하고 커밋을 생성합니다.

커밋 메시지 규칙:

- `feat:`, `fix:`, `docs:`, `style:`, `chore:`, `refactor:`, `test:` 중 하나의 prefix로 시작해야 합니다.
- 반드시 영어로 작성합니다.
- 간결하고 명확하게 작성합니다.

## 4단계: 푸시 및 PR 생성

브랜치를 푸시합니다:

```
git push origin $ARGUMENTS
```

PR을 생성합니다:

- **타이틀**: 영어로 작성, 커밋 메시지와 동일한 prefix(`feat:`, `fix:`, `docs:`, `style:`, `chore:`, `refactor:`, `test:`)로 시작하며 변경사항을 간결하게 요약
- **본문**: 한글로 작성, 무엇을 왜 변경했는지 설명

PR 생성 시 `--assignee @me` 옵션을 포함하여 본인을 assignee로 지정합니다.

PR을 생성하기 전에 타이틀과 본문을 사용자에게 보여주고 승인을 받습니다.
